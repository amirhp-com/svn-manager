SVN Manager Swift · macOS · SwiftUI v1.0.0 (build 1)

Single-window native Mac app — four tabs, dark mode, Apple liquid-glass vibrancy. Wraps your repeated SVN/Git workflow into one-click buttons with saved auth profiles. Every button has a tooltip explaining what it does and shows the exact svn command in a live activity log. Ships as a real .app bundle with a generated icon and a drag-install DMG.

Repo: github.com/amirhp-com/svn-manager · Site: amirhp.com/landing

Context

You repeatedly run the same SVN sequences to manage WordPress.org plugins (status, add, commit, tag, fix MIME types on assets, prune missing files) and occasionally pull a Git repo for plugin source. This app collapses all of that into a single SwiftUI window with four tabs: SVN Folder, SVN URL, SVN Auth, and About. The tab bar is pinned (its position never shifts when switching tabs), each tab is independently scrollable, and auth profiles can be scoped to all folders or to a specific folder + subfolders with one optional default. The Folder tab lets you choose None — use svn internal auth or any matching saved profile.

Updates in this revision

  • Pinned tab bar — leading-aligned with a trailing Spacer and fixedSize() on each pill, so the tab strip never shifts horizontally between selections.
  • Scrollable tabs — every tab body is wrapped in a ScrollView; long action lists or logs scroll instead of resizing the window.
  • About tab — app icon, version + build, copyright, disclaimer, and clickable links to the website and GitHub repo.
  • Tooltips on every action button — hover any button to see what it does and the exact svn command(s) it will run.
  • Live command log — compound actions (Tag, Fix MIME, Remove deleted) stream each $ svn … step as it runs, not as one lumped block at the end.
  • App icon, .app bundle, DMG installer — generated by tools/make_icon.swift and build.sh.
  • Published to GitHub as amirhp-com/svn-manager.

Stack

Swift 5.9 · SwiftUI · macOS 13+. Window background uses NSVisualEffectView(.hudWindow, .behindWindow) for Apple vibrancy/glass; .preferredColorScheme(.dark) forces dark mode; hidden title bar. Subprocesses for svn and git via Process + /usr/bin/env (honors PATH). Auth profiles persisted as JSON to ~/Library/Application Support/SVNManager/auth.json.

Auth Model

Each AuthProfile = { name, username, password, scopePath?, isDefault }.
  • scopePath nil → applies to all folders.
  • scopePath set → applies only when the inspected folder equals it or is a subfolder.
  • isDefault → auto-selected on the Folder tab when its scope matches; only one profile can be default at a time (saving a new default clears the old one).
  • Folder tab picker always offers None — use svn internal auth as the first option, so the system ~/.subversion credentials are used when no profile is chosen.

Build & Run

cd svn-mananger
swift run                          # quick dev run
./build.sh                         # icon + release .app + DMG installer
open "build/SVN Manager.app"       # launch the bundled app
open  build/SVN-Manager-1.0.0.dmg  # open the installer

Requires Xcode 15 / Swift 5.9 / macOS 13+. Needs svn and git on PATH.

Verification

  1. swift build → completes clean (already verified).
  2. swift run → window opens in dark mode with vibrancy background.
  3. Auth tab: add a profile, mark as default, save → file appears at ~/Library/Application Support/SVNManager/auth.json.
  4. URL tab: paste a real plugin URL + folder, click Checkout → log streams output, folder populates.
  5. Folder tab: Browse to that folder, Inspect → SVN pill lights, svn info shown, default auth pre-selected.
  6. Touch a file → Refresh status shows M; Commit… → revision in log.
  7. Tag version 9.9.9-test → verify on server; Delete remote tag → confirm gone.
  8. Drop a .png in assets/, run Fix attachments → svn proplist -v shows svn:mime-type=image/png.
  9. Manually remove a tracked file, click Prune missing → status clears.

Out of Scope (v1)

  • Keychain-backed password storage (current: plain JSON; AuthStore API surface is ready for a swap).
  • Diff viewer / conflict resolution UI.
  • Multi-window or per-repo "saved" Folder shortcuts (use Tab 2 to checkout, then browse on Tab 1).
